## 1. Введение

Плагин **Mist1351/yandex-games-sdk** — это неофициальная, но наиболее полная интеграция Яндекс.Игр SDK для Godot 4.4+. Он покрывает **все** модули платформы: рекламу (баннер, interstitial, rewarded), покупки, лидерборды, отзывы, данные игрока, информацию об устройстве, ярлыки и полноэкранный режим.

Ключевые преимущества перед ручным JS-мостиком и другими плагинами:

- Полная поддержка `async/await` (корутины Godot 4.4+)
- Авто-инициализация SDK (устраняет ошибки порядка загрузки)
- Баннерная реклама «из коробки»
- Поддержка внутриигровых покупок и отзывов
- Демо-сцена для тестирования

## 2. Установка плагина

1. Скачай последний релиз: [https://github.com/Mist1351/yandex-games-sdk/releases/latest](https://github.com/Mist1351/yandex-games-sdk/releases/latest)
1. Распакуй архив. Перенеси папку `addons/yandex_games_sdk` в корень твоего Godot-проекта.
1. Открой проект в Godot → **Project → Project Settings → Plugins**.
1. Включи плагин **YandexGamesSDK** (поставь галочку).
1. После включения в **Project Settings → Globals → Autoload** автоматически добавится глобальная переменная `YandexSDK`.
1. В нижней панели редактора появится вкладка **Yandex Games SDK**.

### Настройка плагина

Во вкладке **Yandex Games SDK** доступны опции:

| Опция | Рекомендация | Описание |
|---|---|---|
| **Auto Init** | ✅ Включить | SDK инициализируется автоматически при старте игры |
| **Auto GameReady** | ✅ Включить | Автоматически вызывает `game_ready()` после инициализации |
| **Debug Mode** | Включить при разработке | Выводит логи SDK в консоль |

> **Почему Auto Init?** Это решает проблему порядка инициализации: твои автозагрузки (GameManager, SaveManager) гарантированно получат уже готовый SDK.

## 3. Настройка экспорта под Web

Для Яндекс.Игр нужен экспортный пресет **Web** со специальной кастомной фичей.

1. **Project → Export → Add → Web**.
1. В настройках пресета:

- **Custom Features** (вкладка Options): пропиши `yandex`
- **Head Include**: оставь пустым (плагин сам встроит SDK)
- **HTML Canvas Resize Policy**: `2` (Adaptive)
- **Focus Canvas on Start**: `true`
- **Experimental Virtual Keyboard**: `false`
1. Название главного файла при экспорте должно быть `index.html`.

Пример правильного пресета из проекта SpaceAttack:

```
[preset.1]
name="Web"
platform="Web"
custom_features="yandex"
html/canvas_resize_policy=2
html/focus_canvas_on_start=true
```

## 4. Инициализация SDK

### Вариант 1: Auto Init включён (рекомендуется)

Плагин сам инициализирует SDK. Твоя задача — дождаться сигнала `init_succeeded`:

```
# В любой сцене или автозагрузке
func _ready() -> void:
    if YandexSDK.is_inited():
        _on_sdk_ready()
    else:
        YandexSDK.init_succeeded.connect(_on_sdk_ready)

func _on_sdk_ready() -> void:
    print("SDK готов к работе")
    # Здесь можно начинать игру
```

### Вариант 2: Ручная инициализация (Auto Init выключен)

```
func _ready() -> void:
    await YandexSDK.init()
    await YandexSDK.player.init()
    await YandexSDK.leaderboard.init()
    await YandexSDK.payments.init()
    print("SDK готов к работе")
```

### Методы жизненного цикла игры (обязательно для модерации)

```
# При старте игровой сессии
func start_gameplay():
    YandexSDK.player.gameplay_start()

# При окончании игровой сессии (Game Over, Victory, пауза)
func stop_gameplay():
    YandexSDK.player.gameplay_stop()
```

> ⚠️ **Важно:** `gameplay_start()` и `gameplay_stop()` — обязательные методы для прохождения модерации. Они сообщают платформе, когда игрок активен в игре, а когда — нет.

## 5. Баннерная реклама

Баннер — это постоянная полоска рекламы. По требованиям Яндекс.Игр, баннер **не должен перекрывать элементы управления и НЕ должен показываться во время активного геймплея**.

### Где показывать баннер

| Место | Когда |
|---|---|
| Главное меню | При входе, на всё время |
| Ангар / Магазин | При входе, на всё время |
| Game Over / Victory | После смерти или победы |
| Экран паузы | Когда игра на паузе |

### Где НЕ показывать баннер

- ❌ Во время геймплея (волны, бой)
- ❌ Поверх кнопок и HUD

### Код

```
# Показать баннер
await YandexSDK.adv.show_banner()

# Скрыть баннер
await YandexSDK.adv.hide_banner()
```

### Пример: баннер в меню и его скрытие при старте игры

```
# menu.gd
func _ready():
    if YandexSDK.is_inited():
        await YandexSDK.adv.show_banner()
    else:
        YandexSDK.init_succeeded.connect(_show_banner_on_init)

func _show_banner_on_init():
    await YandexSDK.adv.show_banner()

func _on_play_pressed():
    await YandexSDK.adv.hide_banner()  # скрываем перед геймплеем
    get_tree().change_scene_to_file("res://game.tscn")
```

## 6. Полноэкранная реклама (Interstitial)

Interstitial — полноэкранная реклама, которая показывается **только в логических паузах**.

### Где можно показывать

- ✅ Главное меню (при первом запуске)
- ✅ После Game Over
- ✅ После победы над финальным боссом
- ✅ При переходе между уровнями
- ✅ На загрузочных экранах

### Где нельзя

- ❌ Между волнами в реальном времени
- ❌ Посреди боя
- ❌ Автоматически без паузы

### Код

```
YandexSDK.adv.show_fullscreen()
```

Метод `show_fullscreen()` не возвращает значение напрямую, но можно отследить события через сигналы:

```
# Доступные сигналы плагина Mist1351:
YandexSDK.adv.show_fullscreen_opened   # реклама открылась
YandexSDK.adv.show_fullscreen_closed   # реклама закрылась (параметр: was_shown: bool)
YandexSDK.adv.show_fullscreen_error    # ошибка показа
```

## 7. Rewarded-реклама (за вознаграждение)

Rewarded-видео — пользователь смотрит рекламу и получает награду (монеты, бустер, жизни и т.д.).

### Код

```
func _ready():
    # Подключаем сигнал награды
    YandexSDK.adv.show_rewarded_video_rewarded.connect(_on_rewarded)

func show_rewarded_ad():
    YandexSDK.adv.show_rewarded_video()

func _on_rewarded() -> void:
    # Пользователь досмотрел рекламу — выдаём награду
    GameManager.add_coins(100)
    print("Награда выдана!")
```

### Где размещать кнопку rewarded-рекламы

- ✅ В магазине («Получить монеты бесплатно»)
- ✅ После смерти («Воскреснуть за рекламу»)
- ✅ В ангаре («Получить бесплатный сундук»)

## 8. Кулдаун и частота показов

По требованиям Яндекс.Игр, минимальный интервал между interstitial-показами — **60 секунд**. Это **обязательное** требование для модерации.

```
var _last_interstitial_time: float = -60.0  # начальное значение позволяет первый показ

func can_show_interstitial() -> bool:
    return Time.get_ticks_msec() / 1000.0 - _last_interstitial_time >= 60.0

func try_show_interstitial():
    if can_show_interstitial():
        _last_interstitial_time = Time.get_ticks_msec() / 1000.0
        YandexSDK.adv.show_fullscreen()
    else:
        print("Кулдаун ещё не прошёл")
```

### Стратегия показа interstitial

Не показывай рекламу при **каждом** Game Over. Лучше — через раз или с кулдауном:

```
var _death_count: int = 0

func on_game_over():
    stop_gameplay()
    _death_count += 1
    
    # Показываем рекламу каждую 3-ю смерть, но не чаще 60 секунд
    if _death_count % 3 == 0 and can_show_interstitial():
        try_show_interstitial()
        await _wait_for_ad_close()
    
    show_game_over_screen()
```

## 9. Управление паузой игры

### Проблема

Реклама **сама управляет паузой** дерева сцены через сигналы `show_fullscreen_opened` / `show_fullscreen_closed`. Если ты тоже управляешь паузой — может возникнуть рассинхрон.

### Правильный подход

**НЕ используй `get_tree().paused` вместе с рекламой.** Вместо этого используй собственный флаг `is_ad_showing` и блокируй игровую логику через него:

```
var is_ad_showing: bool = false

func show_interstitial():
    is_ad_showing = true
    YandexSDK.adv.show_fullscreen_opened.connect(_on_ad_opened)
    YandexSDK.adv.show_fullscreen_closed.connect(_on_ad_closed)
    YandexSDK.adv.show_fullscreen()

func _on_ad_opened():
    # Отключаем игровые таймеры, музыку и т.д.
    AudioServer.set_bus_mute(0, true)

func _on_ad_closed(_was_shown: bool):
    is_ad_showing = false
    AudioServer.set_bus_mute(0, false)
    # Важно: отключаем сигналы, чтобы не накапливались
    YandexSDK.adv.show_fullscreen_opened.disconnect(_on_ad_opened)
    YandexSDK.adv.show_fullscreen_closed.disconnect(_on_ad_closed)
```

### Для rewarded-рекламы

Rewarded не управляет паузой — её нужно обрабатывать вручную:

```
func show_rewarded():
    is_ad_showing = true
    get_tree().paused = true
    YandexSDK.adv.show_rewarded_video_rewarded.connect(_on_rewarded_once, CONNECT_ONE_SHOT)
    YandexSDK.adv.show_rewarded_video_closed.connect(_on_rewarded_closed_once, CONNECT_ONE_SHOT)
    YandexSDK.adv.show_rewarded_video()

func _on_rewarded_closed_once():
    get_tree().paused = false
    is_ad_showing = false
```

> 💡 **Совет:** Используй `CONNECT_ONE_SHOT` при подключении сигналов рекламы, чтобы обработчики не накапливались при повторных показах.

## 10. Архитектура: Создание AdManager

Для чистоты кода рекомендуется создать **единую автозагрузку AdManager**, которая инкапсулирует всю логику рекламы:

```
# ad_manager.gd (autoload)
extends Node

## AdManager — централизованное управление всей рекламой

signal ad_started
signal ad_finished(was_shown: bool)
signal rewarded_ad_rewarded

var is_ad_showing: bool = false
var _last_interstitial_time: float = -60.0
var _interstitial_cooldown: float = 60.0
var _banner_active: bool = false

# ─── БАННЕР ──────────────────────────────────

func show_banner():
    if not _banner_active:
        await YandexSDK.adv.show_banner()
        _banner_active = true

func hide_banner():
    if _banner_active:
        await YandexSDK.adv.hide_banner()
        _banner_active = false

func toggle_banner_for_gameplay(is_gameplay: bool):
    if is_gameplay:
        await hide_banner()
    else:
        await show_banner()

# ─── INTERSTITIAL ─────────────────────────────

func can_show_interstitial() -> bool:
    if is_ad_showing:
        return false
    return Time.get_ticks_msec() / 1000.0 - _last_interstitial_time >= _interstitial_cooldown

func show_interstitial(force: bool = false):
    if not force and not can_show_interstitial():
        return false
    
    is_ad_showing = true
    ad_started.emit()
    _last_interstitial_time = Time.get_ticks_msec() / 1000.0
    
    YandexSDK.adv.show_fullscreen_closed.connect(_on_interstitial_closed, CONNECT_ONE_SHOT)
    YandexSDK.adv.show_fullscreen_error.connect(_on_interstitial_error, CONNECT_ONE_SHOT)
    YandexSDK.adv.show_fullscreen()
    return true

func _on_interstitial_closed(was_shown: bool):
    is_ad_showing = false
    ad_finished.emit(was_shown)

func _on_interstitial_error():
    is_ad_showing = false
    ad_finished.emit(false)

# ─── REWARDED ─────────────────────────────────

func show_rewarded() -> bool:
    if is_ad_showing:
        return false
    
    is_ad_showing = true
    ad_started.emit()
    
    YandexSDK.adv.show_rewarded_video_rewarded.connect(_on_rewarded, CONNECT_ONE_SHOT)
    YandexSDK.adv.show_rewarded_video_closed.connect(_on_rewarded_closed, CONNECT_ONE_SHOT)
    YandexSDK.adv.show_rewarded_video_error.connect(_on_rewarded_error, CONNECT_ONE_SHOT)
    YandexSDK.adv.show_rewarded_video()
    return true

func _on_rewarded():
    rewarded_ad_rewarded.emit()

func _on_rewarded_closed():
    is_ad_showing = false
    ad_finished.emit(true)

func _on_rewarded_error():
    is_ad_showing = false
    ad_finished.emit(false)

# ─── СБРОС КУЛДАУНА (для тестов) ─────────────

func reset_cooldown():
    _last_interstitial_time = -60.0
```

### Использование в игровых сценах

```
# game_over_screen.gd
func show_game_over():
    AdManager.toggle_banner_for_gameplay(false)
    if AdManager.can_show_interstitial():
        AdManager.show_interstitial()
        await AdManager.ad_finished  # ждём закрытия рекламы
    show_ui()

# menu.gd
func _ready():
    AdManager.toggle_banner_for_gameplay(false)

func _on_play_pressed():
    AdManager.toggle_banner_for_gameplay(true)
    get_tree().change_scene_to_file("res://game.tscn")
```

## 11. Сохранение данных игрока

Платформа Яндекс.Игр предоставляет два типа хранилищ:

| Тип | Методы | Макс. размер | Содержимое |
|---|---|---|---|
| **Player Data** | `get_data()` / `set_data()` | 200 КБ | Любые данные (словари, вложенные структуры) |
| **Player Stats** | `get_stats()` / `set_stats()` / `increment_stats()` | 10 КБ | Плоский словарь с числовыми значениями |

### Сохранение комплексных данных (Player Data)

```
# save_manager.gd
func save_game():
    var data = await YandexSDK.player.get_data()
    
    # Обновляем нужные ключи
    data["progress"] = {
        "highest_wave": 10,
        "total_kills": 5234,
        "credits": 15000
    }
    data["unlocked_ships"] = ["vanguard", "phantom"]
    data["achievements"] = {
        "first_blood": true,
        "veteran": true
    }
    
    # Перезаписываем ВСЕ данные на сервере
    await YandexSDK.player.set_data(data)
    print("Игра сохранена")

func load_game():
    var data = await YandexSDK.player.get_data()
    return data  # Dictionary со всеми сохранёнными ключами
```

> ⚠️ **Внимание:** `set_data()` **перезаписывает все данные** на сервере Яндекса. Сначала всегда делай `get_data()`, обновляй нужные ключи, потом сохраняй.

### Сохранение числовых данных (Player Stats)

```
# Подходит для простых значений: монеты, счёт, количество игр
func save_stats(coins: int, high_score: int):
    var stats = await YandexSDK.player.get_stats()
    stats["coins"] = coins
    stats["high_score"] = max(stats.get("high_score", 0), high_score)
    await YandexSDK.player.set_stats(stats)

# Инкрементальное обновление (без get_stats)
func add_coins(amount: int):
    await YandexSDK.player.increment_stats({"coins": amount})
```

### Стратегия сохранения

- **Player Data** — для комплексных данных (прогресс, ачивки, разблокировки)
- **Player Stats** — для простых счётчиков и валют
- Сохраняйся после важных событий: конец волны, покупка, смерть, выход
- Не превышай лимиты (200 КБ / 10 КБ) — иначе сохранение молча откажется работать

## 12. Лидерборды и отправка счёта

### Инициализация

Если авто-инициализация включена, лидерборды готовы сразу. Иначе:

```
await YandexSDK.leaderboard.init()
```

### Отправка счёта

```
func submit_score(score: int):
    await YandexSDK.leaderboard.set_score("main", score)
    print("Счёт отправлен в лидерборд")
```

### Получение записей

```
# Получить запись текущего игрока
func get_my_score():
    await YandexSDK.leaderboard.get_player_entry("main")
    YandexSDK.leaderboard.player_entry_loaded.connect(_on_player_entry, CONNECT_ONE_SHOT)

func _on_player_entry(entry: Dictionary):
    print("Мой рекорд: ", entry.get("score", 0))
    print("Моё место: ", entry.get("rank", "—"))

# Получить топ-10 и 5 записей вокруг игрока
func get_leaderboard():
    await YandexSDK.leaderboard.get_entries("main", true, 5, 10)
    YandexSDK.leaderboard.entries_loaded.connect(_on_entries, CONNECT_ONE_SHOT)

func _on_entries(data: Dictionary):
    for entry in data.get("entries", []):
        print(entry["player"]["publicName"], " — ", entry["score"])
```

### Правильная отправка счёта (без блокировки UI)

```
# game_over.gd
func show_game_over(final_score: int):
    show_ui()  # сначала показываем экран
    _submit_score(final_score)  # потом отправляем счёт в фоне

func _submit_score(score: int):
    await YandexSDK.leaderboard.set_score("main", score)
    print("Счёт отправлен")
```

> 💡 **Почему так?** Если сделать `await` до показа UI, экран Game Over не отобразится, пока не завершится сетевой запрос.

## 13. Внутриигровые покупки

Плагин Mist1351 поддерживает модуль `YandexSDK.payments`.

### Инициализация каталога

В консоли разработчика Яндекс.Игр создай товары, затем в игре:

```
# Авто-инициализация плагина делает это сама.
# При ручной:
await YandexSDK.payments.init()
```

### Покупка товара

```
func buy_item(item_id: String):
    var result = await YandexSDK.payments.purchase(item_id)
    if result:
        print("Покупка совершена: ", item_id)
        grant_item(item_id)
    else:
        print("Покупка отменена или ошибка")

func grant_item(item_id: String):
    match item_id:
        "remove_ads":
            GameData.ads_removed = true
        "coins_500":
            GameData.add_coins(500)
```

### Проверка наличия покупок

```
func check_purchases():
    var purchases = await YandexSDK.payments.get_purchases()
    for purchase in purchases:
        print("Куплено: ", purchase.get("productID"))
        grant_item(purchase.get("productID"))
    # Пометить купленные как потреблённые (consumable)
    await YandexSDK.payments.consume_purchases(purchases)
```

### Проверка каталога товаров

```
func get_catalog():
    var catalog = await YandexSDK.payments.get_catalog()
    for item in catalog:
        print(item["id"], " — ", item["price"], " ", item["priceCurrencyCode"])
```

## 14. Отзывы и рейтинг (Feedback)

Показ окна отзыва помогает повысить рейтинг игры.

### Проверка и запрос отзыва

```
func try_show_review():
    var can_review = await YandexSDK.feedback.can_review()
    if can_review:
        var result = await YandexSDK.feedback.request_review()
        if result == "accepted":
            print("Игрок оставил отзыв — спасибо!")
            # Можно выдать бонус
        elif result == "rejected":
            print("Игрок отказался от отзыва")
    else:
        print("Нельзя показать отзыв сейчас (уже показывали, или нет авторизации)")

```

### Когда запрашивать отзыв

| Ситуация | Шанс на позитивный отзыв |
|---|---|
| После победы над финальным боссом | 🟢 Высокий |
| После 5+ успешных забегов | 🟢 Высокий |
| При первом запуске | 🔴 Низкий |
| После проигрыша | 🔴 Низкий |
| Сразу после покупки | 🟡 Средний |

```
var _games_played: int = 0
var _review_requested: bool = false

func on_victory():
    _games_played += 1
    if _games_played >= 5 and not _review_requested:
        _review_requested = true
        await get_tree().create_timer(2.0).timeout
        try_show_review()
```

> ⚠️ **Ограничение платформы:** Яндекс.Игры позволяет показывать окно отзыва не чаще одного раза за определённый период. `can_review()` сам определяет, можно ли сейчас.

## 15. Авторизация игрока

Многие функции SDK требуют авторизации (сохранения, лидерборды, покупки). Неавторизованный игрок получит ошибки.

### Проверка авторизации

```
func check_auth():
    await YandexSDK.player.check_is_authorized()
    YandexSDK.player.is_authorized_checked.connect(_on_auth_checked, CONNECT_ONE_SHOT)

func _on_auth_checked(is_authorized: bool):
    if is_authorized:
        print("Игрок авторизован — можно сохранять и отправлять счёт")
    else:
        print("Игрок не авторизован — используем локальное сохранение")
```

### Открытие окна авторизации

```
func prompt_login():
    YandexSDK.player.open_auth_dialog()
    # После закрытия диалога проверяем авторизацию
    YandexSDK.player.auth_dialog_closed.connect(_on_auth_dialog_closed, CONNECT_ONE_SHOT)

func _on_auth_dialog_closed():
    check_auth()
```

> 💡 **Лучшая практика:** Не блокируй игру, если игрок не авторизован. Используй локальное сохранение (файл), а при авторизации синхронизируй с сервером.

## 16. Полноэкранный режим

Модуль `YandexSDK.fullscreen` позволяет управлять полноэкранным режимом браузера:

```
# Выйти из полноэкранного режима
await YandexSDK.fullscreen.exit()

# Войти в полноэкранный режим
await YandexSDK.fullscreen.enter()
```

Обычно это не требуется для Web-игр, но может пригодиться для кат-сцен или особых режимов.

## 17. Полный чек-лист перед модерацией

Перед отправкой игры на модерацию в Яндекс.Игры, проверь каждый пункт:

### Инициализация и жизненный цикл

- `game_ready()` вызывается после инициализации SDK (или включён Auto GameReady)
- `gameplay_start()` вызывается при старте игровой сессии
- `gameplay_stop()` вызывается при Game Over, Victory, паузе

### Реклама

- Интерстишл **НЕ** показывается во время активного геймплея
- Интерстишл показывается не чаще 1 раза в 60 секунд
- Баннер **НЕ** показывается во время геймплея
- Баннер не перекрывает элементы управления
- После закрытия рекламы игра не зависает
- После закрытия рекламы игра не продолжается в фоне без управления
- Rewarded-реклама корректно выдаёт награду
- Есть запасной путь, если реклама не загрузилась (ошибка)

### Данные и сохранения

- Счёт сбрасывается при старте новой игры
- Счёт отправляется в лидерборд при Game Over / Victory
- Сохранения не превышают 200 КБ (Player Data) и 10 КБ (Stats)
- При загрузке проверяется наличие сохранений (пустой словарь по умолчанию)

### Покупки и отзывы

- Покупки корректно обрабатываются (выдача товара, потребление consumable)
- Запрос отзыва происходит в уместный момент (после победы, не раньше 5+ игр)

### Технические требования

- Экспортный пресет Web с custom_features = `yandex`
- Главный файл называется `index.html`
- Игра работает через `@yandex-games/sdk-dev-proxy`
- Debug-панель показывает `IT` (инициализация верна)

## 18. Типовые ошибки и их решение

### 1. «Игра зависает при первом показе рекламы»

**Причина:** сигнал закрытия рекламы эмитится синхронно до того, как `await` начинает его ждать.

**Решение:** подключай обработчики **до** вызова метода показа:

```
# ПРАВИЛЬНО
YandexSDK.adv.show_fullscreen_closed.connect(_on_closed, CONNECT_ONE_SHOT)
YandexSDK.adv.show_fullscreen()

# НЕПРАВИЛЬНО
YandexSDK.adv.show_fullscreen()
await _wait_somehow()  # сигнал уже улетел
```

### 2. «Счёт не отправляется в лидерборд»

**Причина:** забыт `await` перед `set_score()`.

**Решение:**

```
# НЕПРАВИЛЬНО
YandexSDK.leaderboard.set_score("main", score)  # без await — пропущено

# ПРАВИЛЬНО
await YandexSDK.leaderboard.set_score("main", score)
```

### 3. «После рекламы игра продолжается в фоне»

**Причина:** реклама сама сняла паузу, которую ты поставил.

**Решение:** не используй `get_tree().paused` одновременно с рекламой. Управляй через `is_ad_showing` флаг (см. раздел 9).

### 4. «Данные не сохраняются на сервере»

**Причина:** превышен лимит в 200 КБ / 10 КБ, или игрок не авторизован.

**Решение:**

- Проверяй размер данных перед отправкой
- Проверяй авторизацию через `check_is_authorized()`
- Не сохраняй каждую секунду — делай паузы между сохранениями

### 5. «Дублирование сигналов рекламы»

**Причина:** обработчики подключаются при каждом показе, но не отключаются.

**Решение:** используй `CONNECT_ONE_SHOT`:

```
YandexSDK.adv.show_fullscreen_closed.connect(_on_closed, CONNECT_ONE_SHOT)
```

### 6. «Ошибка `YaGames is not defined` при локальном тесте»

**Причина:** SDK доступен только на домене Яндекс.Игр или через dev-proxy.

**Решение:** используй `@yandex-games/sdk-dev-proxy` для локальной разработки:

```
npm install -g @yandex-games/sdk-dev-proxy
npx @yandex-games/sdk-dev-proxy -p . --app-id=ТВОЙ_ID -c
```
